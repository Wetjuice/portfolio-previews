Product & UX Context

The challenge: Design a standalone, pre-launch experience that captures player interest, showcases the game's identity, and drives social sharing — without giving away the full game.

Scope: End-to-end character creation UX, community sharing flows, and the onboarding experience for players who had never touched Saints Row before.

Outcome: The demo generated significant pre-release community engagement and social sharing. It served as both a marketing activation and a design validation exercise for the full game's character systems.

Saints Row · Customization UX
Deep Silver Volition · Saints Row
Customization Suite UX

Holistic UX design for all customization suites: Character, Vehicle, Weapon, Crew, and HQ. Goal: make deep, complex feature sets feel accessible and intuitive. Led 10–13 developers.

UX SystemsFeature DesignAccessibilityHUD

UX Approach

The problem: Five distinct customization systems, each with significant depth, all needing to feel cohesive and learnable without sacrificing creative power.

Design approach: Unified information architecture across all suites — consistent navigation patterns, progressive disclosure, and a visual language that communicated system depth without being overwhelming. UX prototyped in Adobe XD before engineering handoff.

Transferability: This work is directly analogous to complex enterprise or consumer product UX — multi-feature systems with mixed-expertise users requiring layered disclosure and consistent mental models.

Research-Driven UX

Process: Used Sony's Player Segments framework — a proprietary player motivation research tool — to identify the target audience's core motivations and frustration patterns.

Application: Player segment insights directly informed information architecture decisions: what to show first, what to hide until earned, how to communicate reward anticipation, and how to frame progression milestones in terms that resonated with each player type.

UX & Tooling

UX work: Designed complete user flows for multiple feature areas including combat, inventory, progression, and store interactions — all within the constraints of small mobile screens and session-based player behavior.

Design-data toolset: Built a custom tool to bridge design intent and implementation — designers could spec ability behavior in structured data format that fed directly into the game system. Eliminated translation errors and significantly reduced iteration time.

This kind of designer-built tooling is a recurring theme: understanding both the design need and the technical constraint well enough to build the bridge between them.

For Non-Games Recruiters

UX is UX. The Domain Changes. The Thinking Doesn't.

Player-first UX in games is among the most demanding UX discipline there is. You're designing for millions of players with wildly different skill levels, zero tolerance for friction, and immediate feedback loops. The empathy, systems thinking, and cross-disciplinary collaboration that makes great game UX — translates directly to SaaS, fintech, enterprise tools, and beyond.
